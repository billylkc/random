** Question one

*Q:* Advice for Non engineering background?

*Answer*
- Focus on *delivery* and not modeling (Application focused. To be covered.)
  - Dashboard
  - API
  - etc,..
- *Quantify* and Measuring uplift
  - Conversion up by x% (comparing to control group), which translate to x money
- Small data flavor to other team
  - Team up with money related department. e.g. Marketing, Finance, etc,..
  - Automate some tasks
- Dont use jupyter notebook
  - Invest in your editor, sharpen your tools.
- Dont be afraid of Linux or command line program
  - Try first
- Learn to *push projects*
  - Make assumption. Calculate ROI.
    - Say email campaign.
    - The model can increase open rate by 50%. You can always derive the ROI from that.
  - Tell me what values the solution can bring.
  - Explain limitation
  - Dont treat DS solution as a black box. Guide the user through your *thought process* or steps.

*Q:* Learn Program or other program first or do project first?

*Answer*:
- Always Project first.
- Find something you can see *progress*. Say what you can see/achieve (in terms of objectives) in two weeks/one month.
  - Can be object detection (e.g. using yolo)
  - Can be slack/telegram bot
  - Can be web scraping
  - etc,..
- *Dont limit yourself to Data Science Project only*. Sometimes it takes time to do a proper DS project.
- Read from any source (Medium, Youtube, Paper, LinkedIn, Github, etc,..) that you find interesting, and match it with your own idea or work related project.
